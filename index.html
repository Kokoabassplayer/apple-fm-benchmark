Field report · on-device · measured

There's a new model inside your Mac. We measured it.

macOS 27 ships an on-device language model you can call from the command line. No API key, no cloud, no per-token bill. This is an open, reproducible benchmark of it — every number below was measured on the machine in the spec strip above, and you can re-measure it on yours.

01 · CONTEXT WINDOW

~4,000 usable tokens — and it actually remembers

The hard ceiling (binary-searched) is ~4k tokens. The honest finding is the second part: facts buried in the middle of documents up to 3,000 tokens long were retrieved every time — 12 of 12. It fits the context and understands it.

Measured (amber): functional ceiling 4,037 tok ok → 4,179 tok hard error; multi-needle recall 3/3 at every length up to 3,000 tok. Reference (grey, vendor-stated, not measured by us): GPT-4o 128k · Claude Sonnet 200k · Gemini 1.5 Pro 1M. Log scale.
02 · THROUGHPUT — TWO SPEEDS

Prefill ~967 tok/s. Decode ~71 tok/s.

A single "tokens/sec" number lies — this model has two. It reads your prompt fast (prefill, ~967 tok/s) but writes the answer slower (decode, ~71 tok/s). We separated them with linear regression, not a single stopwatch.

Fit: ttft = 0.054 + 0.00103·n → slope = prefill rate. r²≈0.99 across 5 input sizes.
Fit: wall = −0.36 + 0.0142·out_tok → decode 70.7 tok/s. Per-length: 75 / 73 / 72 tok/s (monotonic, consistent).
03 · STANDARD ACCURACY SUITES

How smart is it, on tests people recognise?

Vendor marketing numbers are impossible to compare. So we ran the same open suites everyone uses — grade-school math (GSM8K), code generation (HumanEval), and broad knowledge (MMLU). Subset sizes chosen for a ~1-hour run on one Mac.

All greedy (--greedy) for reproducibility. HumanEval executed in an isolated sandbox (timeout 10s, no network). Scores are not compared to cloud models here — different model class — they're a baseline for this on-device model that others can reproduce on the same chip.
04 · NETWORK PRIVACY

During a 24-second generation, zero model-related traffic

We can't prove "offline" with a passive probe (background apps always chatter), so we did the honest thing: measured every byte on the interface during inference vs an equal idle window, and listed which process sent each byte. None of them were the model.

Inference window: 63 KB out · 9 processes. Idle window: 46 KB out · 10 processes. Every process in both windows was an unrelated background app — none was the model. See the evidence →

Honest caveat: a passive probe measures the interface, not the model. The definitive test is disabling Wi-Fi and confirming inference still works — a manual step, not auto-badged. What we can say: no model-related process transmitted, and inference added no traffic beyond idle noise.
05 · WHAT'S IT GOOD FOR?

A decision guide, not a score

Verdicts below are derived from the measured numbers above (speed, context, accuracy, privacy) — not vibes. Green = well-suited, amber = workable, red = not the right tool.

06 · VERDICT

The takeaway

A genuinely useful local model — not a GPT-4 replacement, and it shouldn't be judged as one. Its job is the one it's built for: fast, private, always-on assistance on your own machine.

Strengths: instant cold-ish start, ~0.4s warm first-token, perfect mid-context recall, and the strongest possible privacy story (on-device). Limits: a ~4k context, ~71 tok/s structured-output decode, and accuracy suited to everyday tasks — not frontier reasoning.

07 · METHOD & LIMITATIONS

How to trust (and reproduce) this

  • Spec-labeled. Every number applies to the M3 Pro / 18GB / macOS 27.0 machine in the top strip. A different chip will differ.
  • Regression, not stopwatch. Prefill from ttft-vs-input slope; decode from wall-vs-output slope. Single-point "tokens/sec" is misleading and we don't report it.
  • Open harness. bench/run_all.py re-runs everything. Datasets (GSM8K, HumanEval, MMLU) are downloaded and scored locally; licenses noted in the repo.
  • Measured vs reference split. Amber = we measured it on this Mac. Grey = vendor-stated marketing figures, included for scale, not for comparison of like-for-like.
  • "3B" unconfirmed. Apple does not publish the parameter count; we don't assert it. Everything else is measured directly.
  • Privacy is evidence, not proof. We observed no model traffic; proving fully-offline needs the Wi-Fi-off test.
