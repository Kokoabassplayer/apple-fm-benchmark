## โพสต์ 2 / 7 — มันคืออะไร (แนบ slide2.png)

ตัวที่วัดคือ Apple Foundation Models ใน macOS 27 เรียกผ่าน command line ตัวเดียว: `fm`

- โมเดลอยู่ในเครื่อง ไม่ใช่คลาวด์
- ติดมาฟรีกับเครื่อง ไม่มี subscription
- เป็นมาตรฐานเปิดที่ทำซ้ำได้ — ทุกตัวเลขวัดจากเครื่องจริง และคุณรันบนเครื่องคุณเองได้

clone repo แล้วรันสองคำสั่ง:
`bash bench/spec.sh`  → `python3 bench/run_all.py`

https://github.com/kokoabassplayer/apple-fm-benchmark

## โพสต์ 3 / 7 — บริบท (แนบ slide3.png)

เรื่องที่โฆษณาไม่บอก: โมเดลในเครื่องมี context แค่ ~4,000 โทเคน
ต่อให้ใส่ได้ ก็ไม่แปลว่าจะจำได้

แต่ใน 4k นั้น — ดึงข้อมูลกลางเอกสารได้ 100%
(12/12 needles · ทุกตำแหน่ง · ยาวสูงสุด 3,037 โทเคน)

เราแยกให้ชัด: "พอดี" (binary search → 4,037) vs "จำได้จริง" (multi-needle recall)
เพราะเลขเดียวบอกความจริงไม่ได้

## โพสต์ 4 / 7 — ความเร็ว (แนบ slide4.png)

"tokens/sec" ตัวเดียวโกหกคุณ — โมเดลมีสองความเร็ว:

อ่าน prompt (prefill): ~967 tok/s
เขียนคำตอบ (decode): ~71 tok/s

ทั้งสองค่าหาจาก ถดถอยเชิงเส้น ไม่ใช่จับเวลาจุดเดียว
โทเคนแรกตอนอุ่น ~0.42 วินาที (เย็น ~2.44 วินาทีตอนโหลดครั้งแรก)

ความเร็วอ่านเกินคาดมาก — เร็วกว่าเขียนเกือบ 14 เท่า

## โพสต์ 5 / 7 — ความเป็นส่วนตัว (แนบ slide5.png)

คำถามใหญ่: ข้อมูลรั่วออกเครื่องไหม?

เราดักฟังเครือข่ายระหว่างสร้างคำตอบ 23 วินาที:
→ โพรเซสของโมเดลที่ส่งข้อมูลออก = 0
→ ที่เห็นมีแค่ ChatGPT · Spotify · Codex · mDNSResponder (แอปพื้นหลัง ไม่ใช่โมเดล)

differential en0 bytes อยู่ในระดับ noise (สลับเครื่องหมายทุกครั้งที่รัน)

พิสูจน์ออฟไลน์เต็มรูปแบบ = ปิด Wi-Fi แล้วรันดู (เราทำไม่ได้ให้อัตโนมัติ ต้องทำเอง)

## โพสต์ 6 / 7 — คะแนนจริง (แนบ slide6.png)

คะแนนในเกณฑ์มาตรฐานที่คนรู้จัก (greedy, โมเดลออนดีไวซ์):

🧮 GSM8K (เลขประถม): 81.3%  (244/300)
💻 HumanEval (เขียนโค้ด): 54.9% pass@1  (90/164)
📚 MMLU (รู้รอบ 57 วิชา): 60.0%  (308/513)

ไม่ใช่ GPT-4 แต่สำหรับโมเดลที่รันบนเครื่องเราเอง ฟรี ไม่ส่งข้อมูลออก — ใช้งานได้จริง
