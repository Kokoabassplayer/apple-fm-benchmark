# มาตรฐานวัด Apple Foundation Models — โมเดลออนดีไวซ์

เกณฑ์มาตรฐานที่ทำซ้ำได้ ระบุสเปกเครื่องชัดเจน ของ **โมเดลภาษาออนดีไวซ์** ที่ Apple ใส่มาใน
**macOS 27** (Apple Foundation Models เรียกผ่าน `fm` CLI) ไม่ต้องใช้ API key ไม่ต้องเชื่อมคลาวด์
ไม่มีค่าใช้จ่ายต่อโทเคน — ทุกตัวเลขวัดจากเครื่อง Mac เครื่องเดียว และคุณเองก็วัดบนเครื่องคุณได้

🔗 **แดชบอร์ด:** <https://kokoabassplayer.github.io/apple-fm-benchmark>

> วัดบน **MacBook Pro · Apple M3 Pro · 11 คอร์ (5P+6E) · 18 GB · macOS 27.0 (build 26A5353q)**

## ผลที่วัดได้ (เครื่องนี้)

| | ค่า | วิธีวัด |
|---|---|---|
| **Context ที่ใช้ได้** | ~4,000 โทเคน; ดึงข้อมูลกลางเอกสารได้ 100% ถึง 3k | binary-search ขีดจำกัด + multi-needle recall |
| **Prefill** (อ่าน prompt) | ~967 tok/s | ความชัน TTFT vs ความยาว input |
| **Decode** (เขียนคำตอบ) | ~71 tok/s | ความชันเวลา vs ความยาว output |
| **โทเคนแรก (อุ่น)** | ~0.42 วินาที | มัธยฐาน, output สั้น |
| **โทเคนแรก (เย็น)** | ~2.44 วินาที | ครั้งแรก (โหลดโมเดล) |
| **ความเป็นส่วนตัวเครือข่าย** | ไม่พบโพรเซสของโมเดล | ดิฟเฟอเรนเชียล en0 + nettop ระหว่างสร้างคำตอบ 23 วินาที |
| **GSM8K / HumanEval / MMLU** | ดูในแดชบอร์ด | ชุดทดสอบมาตรฐานเปิด |

ตัวเลข "tokens/sec" ตัวเดียวทำให้เข้าใจผิด เราเลยรายงาน **สอง** ความเร็ว (prefill กับ decode)
แต่ละค่าหาจาก **ถดถอยเชิงเส้น** ไม่ใช่จับเวลาแบบจุดเดียว

## ทำซ้ำบน Mac ของคุณ

ต้องมี: macOS 27 ที่มี `fm` CLI (`/usr/bin/fm`), Python 3.10+ และ `git` (Apple Foundation Models มีเฉพาะ macOS 27+)

```bash
git clone https://github.com/kokoabassplayer/apple-fm-benchmark
cd apple-fm-benchmark
bash bench/spec.sh                 # บันทึกสเปกเครื่องคุณ
python3 bench/run_all.py           # latency + context + privacy + 3 ชุดทดสอบ (~1 ชั่วโมง)
```

แล้วเปิด `index.html` (หรือ commit `results/all.json` ของคุณ แดชบอร์ดจะอัปเดตอัตโนมัติ)

> **HumanEval รันโค้ดที่โมเดลสร้าง** ใน subprocess แยก (timeout 10 วินาที, temp dir ใหม่, ไม่มีเครือข่าย
> ไม่มีสิทธิ์พิเศษ) ดู `bench/DATA_LICENSES.md`

## ทำไมถึงน่าเชื่อถือ

- **ระบุสเปกชัดเจน** — ทุกตัวเลขผูกกับเครื่องในแถบสเปก ชิปอื่นจะต่างออกไป
- **ถดถอยเชิงเส้น ไม่ใช่จับเวลา** — prefill จากความชัน ttft-vs-input, decode จากความชัน wall-vs-output
- **Context = พอดี vs ดึงได้จริง** — ขีดจำกัด (พอดี) + multi-needle recall (ดึงข้อมูลกลางเอกสารได้จริงหรือไม่)
- **ความเป็นส่วนตัวคือหลักฐาน ไม่ใช่ข้อพิสูจน์** — วัดดิฟเฟอเรนเชียล + ลิสต์โพรเซส nettop ระหว่าง inference vs idle
  ไม่พบโพรเซสของโมเดล; การพิสูจน์ออฟไลน์เต็มรูปแบบต้องปิด Wi-Fi (ขั้นตอน manual)
- **แยก วัดเอง vs อ้างอิง** — อำพัน=วัดบนเครื่องนี้ · เทา=ตัวเลขการตลาดของผู้ผลิต ใส่ไว้เป็นมาตราสวน
- **ไม่ยืนยัน "3B"** — Apple ไม่เปิดเผยจำนวนพารามิเตอร์
